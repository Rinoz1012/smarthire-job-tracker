# SmartHire — Step by Step GitHub Commit Guide

Follow these steps IN ORDER. Each step = one commit.
This gives you a clean, realistic commit history that looks professional.

---

## FIRST TIME SETUP (do this once)

```bash
# 1. Download and unzip the project files I gave you
# 2. Open terminal and go into the folder
cd smarthire-job-tracker

# 3. Initialize git
git init

# 4. Go to github.com → New Repository
#    Name: smarthire-job-tracker
#    Visibility: Public
#    Do NOT add README (we have our own)
#    Click: Create repository

# 5. Connect your local folder to GitHub
git remote add origin https://github.com/YOUR_USERNAME/smarthire-job-tracker.git
```

---

## COMMIT 1 — Project setup

```bash
# Only add these files for the first commit
git add .gitignore
git add README.md

git commit -m "Initial project setup with README and gitignore"
git push -u origin main
```

## AFTER ALL COMMITS — Deploy to Vercel

1. Go to vercel.com
2. Import your GitHub repo
3. Set root directory to: frontend
4. Add environment variable: REACT_APP_ANTHROPIC_API_KEY = your_key
5. Click Deploy
6. Copy the live URL and paste it into README.md

```bash
# After updating README with your live URL:
git add README.md
git commit -m "Add live demo link to README"
git push
```

## IF YOU GET ERRORS

Error: "failed to push, rejected"
Fix:
git pull origin main --rebase
git push

Error: "remote origin already exists"
Fix:
git remote remove origin
git remote add origin https://github.com/YOUR_USERNAME/smarthire-job-tracker.git

Error: "npm not found"
Fix: Install Node.js from nodejs.org first
